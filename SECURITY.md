# Security Policy

## サポート対象バージョン

Nudge は v1.0 以前の開発フェーズにあり、最新の `main` ブランチのみをセキュリティ修正の対象としています。古いバージョンを利用している場合は最新版へのアップグレードを推奨します。

| バージョン | サポート |
|---|---|
| `main`（最新リリース） | ✅ |
| それ以前 | ❌（自身でフォーク・パッチ適用してください） |

`v1.0` リリース以降は、サポート対象バージョンを明記します。

## 脆弱性の報告

**公開 Issue では報告しないでください。** 攻撃者に悪用される前に修正・周知できるよう、以下のいずれかの非公開チャネルで連絡してください：

1. **推奨**: [GitHub Security Advisories](https://github.com/tetsukame/nudge/security/advisories/new) で「Report a vulnerability」を作成
2. メール: `tetsu.kame@gmail.com`（PGP 鍵未整備のため、機微な詳細は最小限に）

報告には以下を含めていただけると調査がスムーズです：

- 影響範囲（どの機能・どのバージョン）
- 再現手順（最小 PoC）
- 想定される被害（情報漏洩 / 権限昇格 / DoS など）
- 報告者の連絡先（公開可否）

## レスポンス目標

ベストエフォートでの対応となりますが、以下を目標とします：

- 受付の確認: 3 営業日以内
- 一次対応・影響範囲調査: 7 営業日以内
- パッチ公開: 重大度に応じ 14〜30 日以内
- 公開 advisory: 修正リリース後

## 既知のセキュリティ考慮事項

利用者がセルフホスト時に意識すべき点を列挙します：

### 1. `IRON_SESSION_PASSWORD` のローテーション

セッション cookie とデータベース内の暗号化フィールド（通知設定の SMTP パスワード等）を派生キーで保護しています。**この値を変更すると、全ユーザーのセッションが無効化され、暗号化済みのフィールドが復号できなくなります**。ローテーションする場合は事前に DB の暗号化済みフィールドをマイグレーションする必要があります（v1.0 までに自動マイグレーション機構を整備予定）。

最低 32 文字のランダム値を `openssl rand -base64 32` 等で生成して固定してください。

### 2. PostgreSQL ロール分離

このアプリは 2 つの DB ロールを使い分けます：

- `DATABASE_URL_ADMIN`: マイグレーション・管理用。superuser 相当の権限が必要
- `DATABASE_URL_APP`: アプリケーション実行時。`nudge_app` ロールで RLS 強制

**本番環境では絶対に `DATABASE_URL_APP` に superuser を指定しないでください。** RLS が `BYPASSRLS` 属性で素通しされ、テナント分離が破られます。

### 3. Keycloak Service Account の最小権限

ユーザー / 組織 / グループの同期に使う Keycloak service account には、以下の必要最小限のロールのみ付与してください：

- `view-users`, `view-realm`, `view-groups`

**`manage-users` 等の書き込み権限は不要です**。同期は read-only の操作です。

### 4. `SYNC_API_KEY`

cron 等から同期エンドポイントを叩く場合の API キーです。32 文字以上のランダム値を推奨。漏洩した場合は速やかにローテーションし、cron 設定も更新してください。

### 5. テナント間の URL 越境

URL は `/t/<tenant_code>/...` 形式ですが、ログイン済みセッションの `tenantId` で常に検証されます。別テナントのコードを入れても、自分のテナント以外のデータは取得・更新できないことを RLS テストで継続的に検証しています（`tests/rls/`）。

### 6. 通知チャネル設定の暗号化

SMTP パスワード、Teams Webhook URL、Slack Webhook URL は DB 内で `IRON_SESSION_PASSWORD` 派生キーで AES-256-GCM 暗号化して保存しています。**DB バックアップを取得する際は、`IRON_SESSION_PASSWORD` が無いと復号できない点に留意してください**。バックアップとパスワードの保管先は分離することを推奨します。

### 7. CSP / フレーム埋め込み

将来的に Microsoft Teams / SharePoint への iframe 埋め込みを検討していますが、デフォルトでは `frame-ancestors` を制限していません。本番環境ではリバースプロキシ等で適切な CSP ヘッダを付与してください。

## 依存ライブラリの脆弱性

`pnpm audit` で定期的に確認することを推奨します。`v0.17` 以降で dependabot による自動更新を整備予定です（[NDG-5 Phase 5c]）。

## 謝辞

セキュリティ報告にご協力いただいた方には、希望に応じてリリースノートおよび advisory に謝辞を記載させていただきます。
